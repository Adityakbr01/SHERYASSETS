## Setup in 5 minutes

SheryAssets gives you backend features for:

- Authentication and tenant switching
- Membership and role management
- Plan and billing workflows
- API key management for server-to-server access

## Before you begin

- A running server instance
- Base URL for your environment
- A valid user account

<Note>Use local base URL http://localhost:5000 for development.</Note>

## Step 1: Authenticate

Do auth in this order:

1. Send OTP
2. Register with OTP
3. Login
4. Switch tenant if needed

For full details, open /authentication.

### Expected

- You receive access and refresh tokens
- You can call /api/v1/auth/me and get user data

### If you got errors

- 400 means payload format is wrong
- 401 means token is missing or invalid
- 409 means email already exists

Fix from /authentication.

## Step 2: Choose auth strategy

Use one of these for protected endpoints:

- Bearer access token
- Access token cookie

Tenant-scoped routes also need tenant context from activeTenantId in JWT or X-TENANT-ID header.

## Copy-paste setup (recommended)

Set these once, then copy-paste commands.

```bash
BASE_URL=http://localhost:5000
ACCESS_TOKEN=replace_me
ADMIN_TOKEN=replace_me
TENANT_ID=replace_me
PLAN_ID=replace_me
KEY_ID=replace_me
```

Quick sanity checks:

```bash
curl "$BASE_URL/swagger.json"
curl "$BASE_URL/api/v1/health"
curl "$BASE_URL/api/v1/auth/me" -H "Authorization: Bearer $ACCESS_TOKEN"
```

Expected results:

- `/swagger.json` returns OpenAPI JSON
- `/api/v1/health` returns healthy response
- `/api/v1/auth/me` returns current user data when token is valid

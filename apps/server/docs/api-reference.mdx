## Overview

Use this page to call APIs fast with default `curl` commands.

You get:

- Copy-paste API calls
- Example success responses
- Example edge-case responses
- Link to full generated OpenAPI

Works from any stack because HTTP behavior is same for all clients.

  Full endpoint pages (with cURL tabs, copy button, response schema, and edge-case
  responses) are auto-generated from OpenAPI for all modules in the API Reference sidebar
  section.
</Info>

## Quick setup

Set these once:

<Note>
  In the sidebar, open the grouped API sections: API Auth, API Billing, API Tenant, API
  Membership, API ApiKey, and API Plan.
</Note>

```bash
BASE_URL=http://localhost:5000
ACCESS_TOKEN=replace_me
ADMIN_TOKEN=replace_me
TENANT_ID=replace_me
PLAN_ID=replace_me
KEY_ID=replace_me
```

Quick health checks:

```bash
curl "$BASE_URL/api/v1/health"
curl "$BASE_URL/swagger.json"
```

Expected:

- Health returns success response
- Swagger returns OpenAPI JSON

## API map

| Module     | Base path           | Description                                                 |
| ---------- | ------------------- | ----------------------------------------------------------- |
| Auth       | /api/v1/auth        | Registration, login, refresh, profile, and tenant switching |
| Tenant     | /api/v1/tenants     | Tenant discovery and lookup                                 |
| Membership | /api/v1/memberships | Team invites, acceptance, and member management             |
| Billing    | /api/v1/billing     | Subscription intent, payment verification, webhooks         |
| ApiKey     | /api/v1/api-keys    | API key create, list, revoke                                |
| Plan       | /api/v1/plans       | Public plan listing and admin CRUD                          |

## Security map

- Bearer token: Authorization header
- Cookie auth: accessToken cookie
- API key header: x-api-key for service-level integrations

## Copy-paste calls by module

### 1. Auth: login

```bash
curl -X POST "$BASE_URL/api/v1/auth/login" \
	-H "Content-Type: application/json" \
	-d '{"email":"aaditya@sheryassets.com","password":"Password123!"}'
```

Success response (example):

```json
{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": { "_id": "...", "email": "aaditya@sheryassets.com" },
    "accessToken": "eyJ...",
    "refreshToken": "eyJ...",
    "tenants": []
  }
}
```

Edge case (wrong password):

```json
{
  "success": false,
  "message": "Invalid email or password"
}
```

### 2. Tenant: my tenants

```bash
curl "$BASE_URL/api/v1/tenants/my-tenants" \
	-H "Authorization: Bearer $ACCESS_TOKEN"
```

Success response (example):

```json
{
  "success": true,
  "message": "Tenants fetched successfully",
  "data": [{ "_id": "...", "name": "SheryAssets Workspace", "role": "owner" }]
}
```

Edge case (missing token):

```json
{
  "success": false,
  "message": "Unauthorized: access token missing"
}
```

### 3. Membership: invite user

```bash
curl -X POST "$BASE_URL/api/v1/memberships/invite" \
	-H "Authorization: Bearer $ACCESS_TOKEN" \
	-H "X-TENANT-ID: $TENANT_ID" \
	-H "Content-Type: application/json" \
	-d '{"email":"teammate@example.com","role":"member"}'
```

Success response (example):

```json
{
  "success": true,
  "message": "Invitation email sent successfully",
  "data": null
}
```

Edge case (cooldown):

```json
{
  "success": false,
  "message": "You are sending invitations too fast. Please wait a moment."
}
```

### 4. API Keys: create key

```bash
curl -X POST "$BASE_URL/api/v1/api-keys" \
	-H "Authorization: Bearer $ACCESS_TOKEN" \
	-H "X-TENANT-ID: $TENANT_ID" \
	-H "Content-Type: application/json" \
	-d '{"name":"Production Upload Key"}'
```

Success response (example):

```json
{
  "success": true,
  "message": "API key generated. Save this key — it will not be shown again.",
  "data": {
    "id": "...",
    "name": "Production Upload Key",
    "apiKey": "shry_..."
  }
}
```

Edge case (plan limit):

```json
{
  "success": false,
  "message": "API key limit reached (2 keys for Basic plan)",
  "errorCode": "PLAN_LIMIT_API_KEYS"
}
```

### 5. Billing: start subscription

```bash
curl -X POST "$BASE_URL/api/v1/billing/subscribe/$PLAN_ID" \
	-H "Authorization: Bearer $ACCESS_TOKEN" \
	-H "X-TENANT-ID: $TENANT_ID" \
	-H "Content-Type: application/json" \
	-d '{}'
```

Success response (example):

```json
{
  "success": true,
  "message": "Subscription initiated successfully",
  "data": {
    "orderId": "order_Q31k8jx9WJ9V0a",
    "subscriptionId": "...",
    "amount": 2900,
    "currency": "INR"
  }
}
```

Edge case (basic plan):

```json
{
  "success": false,
  "message": "Basic plan cannot be subscribed in this manner"
}
```

### 6. Plans: list all plans (public)

```bash
curl "$BASE_URL/api/v1/plans"
```

Success response (example):

```json
{
  "success": true,
  "message": "Plans fetched successfully",
  "data": [{ "code": "basic", "priceMonthly": 0 }],
  "meta": { "isCache": false }
}
```

Admin edge case (create plan with non-admin token):

```json
{
  "success": false,
  "message": "Forbidden: requires one of system roles [admin]"
}
```

## Common status codes

- `200` success
- `201` created
- `400` bad request or validation error
- `401` auth missing/invalid
- `403` role or tenant permission denied
- `404` resource not found
- `409` conflict (example: duplicate email)
- `429` cooldown/rate limit
